A note on Independent and Dependent Variables for this exercise¶

  • Considering the web traffic metrics, in 'cause and effect' terms for this exercise; Users can be considered as the independent variable because data for the other dependent metrics wouldn't exist if it weren't for the Users.

  • This mean Users is being used as the predictor (X) in order to determine other metrics (y)

  • However, in real-world terms, independent variables should refer to a metric which can be controlled e.g. 'Monthly Campaign Investment'. The reality is that the number of Users engaging with a specific website in a month isn't pre-determined!

  • Technically, the number of Users would actually be dependent on other independent variables.

Understanding Prerequisites¶

  • At the top level, we want to understand how metrics relate to each other.
  • As presented earlier, questions can be answered such as "How many Sessions and Pageviews can be expected if there are x number of Users?".
  • However, in order to get there with help of Linear Regression, various prerequisites are required.

Prerequisites: 1) Linearity and 2) Normal Distribution¶

  • The relationship between independent and dependent variables should be linear
  • The way datapoints are distributed for metrics should conform to a normal distribution

With this in mind, an initial pairplot analysis can be run to test these points, with support of pandas and seaborn. Beforehand, adjustments can be made to the web_traffic_df pandas dataframe to accommodate this.

Prerequisite: 3) Multicollinearity¶

  • Multicollinearity should be kept to a minimal

Pearson's standard correlation coefficient is one of several tests for measuring Multicollinearity

  • High multicollinearity refers to the case whereby variables correlate with each other to such an extent that they overfit the Pearson's multiple regression model.
  • The closer the coefficient values are to 1, the greater the correlation is between the variables. However, values greater than 1 indicate multicollinearity.

Using seaborn FacetGrid with Heat Maps¶

  • Seaborn and pandas can be used to produce a matrix to show the Pearson's standard correlation coefficient between metrics. Heat maps show a distinction for scenarios where a) All Segments are considered and b) Returning Users are excluded.
  • A new dataframe; 'all_and_new_users_only' is produced which considers scenario b)
  • A function is set up to map heatmaps to a matplotlib figure
all_and_new_users_only = web_traffic_df_copy[(web_traffic_df_copy['Segment']=='All Users') 
                                             | (web_traffic_df_copy['Segment']=='New Users')]

def create_heatmap(df, title, pyplot_no):
    #pyplot_no refers to subplot location e.g. 121 refers to 1st row, 2 columns and plot number 1 (top left) 
    # https://stackoverflow.com/questions/1616427/trying-to-position-subplots-next-to-each-other
    plt.subplot(pyplot_no)
    ax = sns.heatmap(df.corr(), annot=True, linewidths=.5, vmin=-0.8, vmax=1, cmap="BuPu")
    ax.set_title(title)
    ax.set_xticklabels(ax.get_xticklabels(), rotation=30)
    ax.set_yticklabels(ax.get_yticklabels(), rotation=30)

plt.figure(figsize=(17, 4.5))
create_heatmap(web_traffic_df_copy, 'Scenario 1: All Segments\n', 121)
create_heatmap(all_and_new_users_only, 'Scenario 2: All Users and New Users\n', 122)

Heatmap findings¶

  • Further to Pairplot findings, Pearson's standard coefficient values also support a high correlation between Users and the dependent variables Page Views and Sessions for both plots.
  • Pageviews and Sessions also show a strong correlation
  • On Multicollinearity: Whilst the coefficient values are less than 1 for pairwise comparisons between Users, Page Views and Sessions, the risk of multicollinearity appears to have lesser risk for the Scenario 2 dataframe .
  • In relation to the Scenario 1 dataframe, the coefficient value for between Users and Sessions is particularly high (0.97), indicating a greater multicollinearity risk

Summary of findings¶

  • At this point, it has been identified proceeding to work with the Scenario 2 dataframe; all_and_new_users_only, helps ensure prerequisites for linear regression analysis thus far are being met.
  • Users, Sessions and Page Views all have a linear relationship, are normally distributed and in Scenario 2 have a reduced risk of Multicollinearity
  • However, please note reducing a considerble number of rows (aka entries) means there are not as many datapoints for the linear regression model. It goes without saying, the more data the better, particularly for the real world! This concerns accuracy trade-offs.
  • Removing Returning Users, means a third of the dataframe has been lost (130 rows down from 195 rows):
