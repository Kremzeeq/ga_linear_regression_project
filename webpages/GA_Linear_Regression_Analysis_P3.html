Web Traffic Data Exploration, with Linear Regression Analysis¶

Part 3: Running Linear Regression with sklearn¶

Following on from the summary of findings, the Scenario 2 dataframe (all_and_new_users_only) with the segments All Users and New Users will be used for Linear Regression models.

The Users column will be used as a predictor (X) for Sessions and Page Views as columns (y) in two separate Linear Regression Models. This is so predictions can be more easily made, based off users.

For each model:

  • A test size of 40% of the 130 data points can be split for the test set (x_test and y_test)
  • The remaining 60% would be used to train the model
  • Please note in the real world the more data the better! this sample size may seem incredibly small
  • The training set is then be fit to the Linear Regression model

Please note:

  • lm_1 refers to the model fit with Users (X) and Sessions (y)
  • lm_2 refers to the model fit with Users (X) and Page Views (y)

Autocorrelation¶

Running the Durbin-Watson test on the y_test and y_pred residuals can provide an indication of whether the linear relationship presented earlier is auto-correlated. Absence of this autocorrelation is a requirement for the linear regression model.

Connor Johnson's code for running the Durbin-Watson test based on the statsmodel python library for linear regression, has been adapted so that the test can be run based on residuals from the sklearn version used herein:

  • First of all residuals are derived for y_test and y_pred in a numpy array.
  • The ravel function removes excessive squared brackets ([ ]) which need to be removed for the Durbin Watson function below.
  • Please refer to online resources for further explanations on Durbin-Watson and numpy
import numpy as np

residuals = np.array(y_test-y_pred).ravel()

def get_durbin_watson(residuals):
    squared_residuals = np.square(residuals)
    sum_of_squared_residuals = np.sum(squared_residuals)
    DW = np.sum( np.diff(residuals )**2.0 ) / sum_of_squared_residuals
    return round(DW, 5)

print("Durbin-Watson result:", get_durbin_watson(residuals))

import numpy as np

def get_durbin_watson_report(y_test, y_pred, title_tail_end):
    residuals = np.array(y_test-y_pred).ravel()
    squared_residuals = np.square(residuals)
    sum_of_squared_residuals = np.sum(squared_residuals)
    DW = np.sum( np.diff(residuals )**2.0 ) / sum_of_squared_residuals
    rounded_result = round(DW, 5)
    print("Durbin-Watson Report: Users vs {}".format(title_tail_end))
    print("Result: {}".format(rounded_result))

Regression Evaluation Metrics¶

Further to validating the linear regression model, there are metrics that can be imported from sklearn to evaluate the error associated with the model. These metrics reflect loss in the model and it can be adjusted to help minimise this and become more accurate.

from sklearn import metrics

def get_regression_evaluation_metrics_report(y_test, y_pred, title_tail_end):
    print("Regression Evaludation Metrics Report: {}\n".format(title_tail_end))
    mean_absolute_error = metrics.mean_absolute_error(y_test, y_pred)
    mean_squared_error = metrics.mean_squared_error(y_test, y_pred)
    root_mean_squared_error = np.sqrt(mean_squared_error)
    print('Mean Average Error (MAE):', mean_absolute_error)
    print('Mean Squared Error (MSE):', mean_squared_error)
    print('Root Mean Squared Error (RMSE):', root_mean_squared_error)

Evaluation metric findings¶

Metrics have been assessed with reference to 'An Introduction to Statistical Learning with applications for R' (James et al. 2015) and also numpy documentation.

Mean Average Error¶

  • This represents a weighted average of all output errors, where zero represents the best outcome and the result shows a high value.

Mean Squared Error¶

  • This metric provides an indication of the extent to which the predicted observation (y_pred) is close to the true observation (y_test).
  • The mean squared value is very high for both models, particularly for lm_2, inferring that the difference between y_pred and y_test may differ substantially for some observations. A judgement can be made that the quality of fit is poor for both models.
  • Thus in real-word terms, relying on these models for making predictions would not be recommended. This further emphasises the case for further data points for the model.

Root Mean Squared Error¶

  • This represents the squared-root of the MSE and has the same units as what is being predicted i.e. (Sessions and Page Views) and appears easier to interpret.
  • With this in mind, a sway of 8684 Sessions and 83109 Page Views still represent a large variance and still pushes the case that the quality of fit for the models can be improved.

Conclusions on Linear Regression Model findings¶

As the saying goes 'there's always room for improvement' and it goes without saying for the linear regression models presented herein. In hierarchial terms, foremost, it would be ideal to obtain more data points for the data set; whether that be adding further historical years or even further dimensions like 'device' to the Google analytics report. However, it's important to be aware that further segmentation of the data can lead to skew effects as was found for the Returning Users segment which was excluded from the models.

At a second tier, once the data has been split into training and testing data sets, it may be advisable to use histograms to check whether data still conforms to a normal distribution. This can help provide an indication of whether enough data points have been sampled in the first place. Moving on from establishing that the data still retains a normal distribution, it should then be posible to evaluate for further tests with more confidence in the model.

Whilst evidence of autocorrelation was found, regression evaluation metrics indicated that there was a high degree of variance between predicted and true observations for y. This brings things full circle to the need for further exploration with more data points.
